# Ternary-Zero Core: W2A16 GEMV Kernel Implementation Plan

## Project Overview
A high-performance 2-bit Weight / 16-bit Activation GEMV kernel for RTX 4060, using ternary weights {-1, 0, 1} with PTX-level bit manipulation and a Rust orchestration layer.

## Phase 2: CUDA Kernel — `kernel/ternary_zero.cu`

### Step 2.1: 2-Bit Packing & Memory Layout

**Encoding Scheme:**
| Ternary Value | 2-bit Code | Notes |
|---|---|---|
| 0  | `00` | Zero-gate (skip) |
| +1 | `01` | Add activation |
| -1 | `10` | Subtract activation |
| 11 | (unused) | Invalid / sentinel |

**Packing:**
- 16 ternary weights → 1 `uint32_t` (32 bits / 2 bits each)
- Bit layout: `[w0:lsb | w1 | w2 | ... | w15:msb]`

**Column-Major Tiled Layout:**
```
// Weight matrix W[M, N] stored as packed uint32_t
// Tile: 64 rows × 1024 cols (packed into 64 × 64 uint32_t tiles)
// Each uint32_t holds 16 consecutive weights along N
// Memory access pattern: threads in a warp read consecutive uint32_t → coalesced 128-byte
```

### Step 2.2: PTX Inline Assembly — Bit-Level Decompression

**Key PTX Instructions:**
1. **`PRMT` (Permute Byte):** Rearrange bytes within a register to position 2-bit fields for extraction
2. **`LOP3.LUT` (3-input Logic):** Apply arbitrary 3-input boolean function via lookup table for masking/mixing bits
3. **BFI / SHL / SHR:** Bit-field insert and shifts for extraction

**Decompression Algorithm (16 weights from uint32_t):**
```
// Input: uint32_t packed (16 weights × 2 bits)
// Strategy: Process 4 weights per iteration using mask + shift
// Use PRMT to pre-swap bytes into convenient positions
// Use LOP3.LUT to simultaneously mask 2-bit fields and classify {00,01,10}

asm volatile(
    "prmt.b32 %0, %1, 0, 0x3210;"   // byte-permute for alignment
    : "=r"(permuted) : "r"(packed));

// For each 2-bit weight w[i]:
//   sign[i] = (w[i] >> 1) & 1    — bit 1: sign flag (1 = negative)
//   mag[i]  = w[i] & 1            — bit 0: magnitude (1 = non-zero)
//   skip[i] = (w[i] == 0)         — both bits zero

// LOP3.LUT truth table for (sign, mag, 0) → accumulate_decision:
//   00 → skip (zero weight)
//   01 → add  (positive)
//   10 → sub  (negative)
//   11 → invalid
```

**Register Budget:**
- ~8 regs for packed weights
- ~8 regs for unpacked sign/mag masks
- ~8 regs for activation staging
- ~8 regs for accumulators
- ~8 regs for temporaries/addresses
- Total: ~40 registers → within 32-64 target, enables 2+ blocks/SM

### Step 2.3: Zero-Gate Arithmetic Logic

```cuda
// Instead of: acc += (float)weight * activation;  (standard FMA)
// Use branchless integer conditional:

// Decode 2-bit weight into (sign_bit, nonzero_bit)
// sign = (weight_bits >> 1) & 1
// nz   = weight_bits & 1

// Conditional accumulation (branchless):
// acc = sign ? (acc - act) : (acc + act)   when nz=1
// acc = acc                                 when nz=0 (zero-gate, skip)

// Implementation with predicated execution:
//   int mask_nz = -nz;           // 0xFFFFFFFF if non-zero, 0x00000000 if zero
//   int mask_neg = -sign;        // 0xFFFFFFFF if negative
//   half2 delta = mask_neg ? -act : act;
//   acc += (delta & mask_nz);    // zero-gated add
```

### Step 2.4: Kernel Signature & Grid Configuration

```cuda
extern "C"
cudaError_t ternary_zero_gemv_f16(
    const uint32_t* __restrict__ weights,   // Packed 2-bit weights [M × (N/16)]
    const __half*   __restrict__ activations,// FP16 activations [N]
    __half*         __restrict__ output,     // FP16 output [M]
    int M, int N,
    cudaStream_t stream
);
```

**Grid/Block Dimensions:**
- Each block handles one output row (one row of W × full activation vector)
- Block size: 256 threads (8 warps)
- Each warp handles N/(16×256) iterations, processing 16 weights per thread per iteration
- Grid: `(M, 1, 1)` — embarrassingly parallel across output rows

### Step 2.5: Shared Memory & L2 Cache Strategy

**Shared Memory Usage:**
- Stage activation tile: `__shared__ __half s_act[TILE_N]` where `TILE_N = 1024`
- Each block loads a 1024-element activation tile cooperatively into shared memory
- All warps in the block read from smem during computation → 8× reduction in global mem reads
- smem usage: 1024 × 2 bytes = 2 KB per block (well within 48 KB limit)

**L2 Cache Policy (Persistent Tiles):**
```cuda
// Use cudaStreamSetAccessPolicy or cudaAccessPolicyWindow
// Pin the most frequently accessed weight rows in L2
cudaStreamAttrValue attr;
attr.accessPolicyWindow.base_ptr  = (void*)weights;
attr.accessPolicyWindow.num_bytes = l2_pin_size;  // e.g., 24 MB (full L2)
attr.accessPolicyWindow.hitRatio  = 1.0f;
attr.accessPolicyWindow.hitProp   = cudaAccessPropertyPersisting;
attr.accessPolicyWindow.missProp  = cudaAccessPropertyStreaming;
cudaStreamSetAttribute(stream, cudaStreamAttributeAccessPolicyWindow, &attr);
```

### Step 2.6: Warp-Level Reduction

```cuda
// After each warp accumulates its partial sum (half2 in registers):
// Step 1: __shfl_down_sync within warp (5 butterfly steps for warp=32)
half2 val = warp_accumulator;
for (int offset = 16; offset >= 1; offset >>= 1) {
    val += __shfl_down_sync(0xFFFFFFFF, val, offset);
}
// Step 2: First lane of each warp writes to shared memory
if (lane_id == 0) s_warp_sums[warp_id] = val;
__syncthreads();

// Step 3: First warp reduces across warps (8 warps → 3 iterations)
if (warp_id == 0) {
    half2 sum = (lane_id < num_warps) ? s_warp_sums[lane_id] : __float2half(0.0f);
    for (int offset = 4; offset >= 1; offset >>= 1)
        sum += __shfl_down_sync(0xFFFFFFFF, sum, offset);
    if (lane_id == 0) output[row] = sum;
}
```

### Step 3.2: `src/bitlinear.rs` — BitLinear Abstraction

```rust
use std::marker::PhantomData;
use half::f16;

pub struct GpuBuffer<T> {
    ptr: *mut T,
    len: usize,
}

impl<T> GpuBuffer<T> {
    pub fn alloc(len: usize) -> Result<Self, CudaError> { /* cudaMalloc */ }
    pub fn copy_from_host(&mut self, data: &[T]) -> Result<(), CudaError> { /* cudaMemcpy H2D */ }
    pub fn copy_to_host(&self, data: &mut [T]) -> Result<(), CudaError> { /* cudaMemcpy D2H */ }
}

impl<T> Drop for GpuBuffer<T> {
    fn drop(&mut self) { /* cudaFree */ }
}

/// Safety: GpuBuffer owns unique device memory
unsafe impl<T> Send for GpuBuffer<T> {}
unsafe impl<T> Sync for GpuBuffer<T> {}

pub struct BitLinear {
    packed_weights: GpuBuffer<u32>,   // [M × ceil(N/16)]
    bias: Option<GpuBuffer<f16>>,     // optional bias [M]
    stream: CudaStream,
    m: usize,
    n: usize,
}

impl BitLinear {
    pub fn new(m: usize, n: usize) -> Result<Self, CudaError> {
        assert!(n % 16 == 0, "N must be multiple of 16 (2-bit packing)");
        let packed_len = m * (n / 16);
        let packed_weights = GpuBuffer::alloc(packed_len)?;
        let stream = CudaStream::new()?;
        Ok(Self { packed_weights, bias: None, stream, m, n })
    }

    /// Pack ternary weights {-1, 0, 1} into 2-bit packed uint32_t format
    pub fn load_weights(&mut self, ternary_weights: &[i8]) -> Result<(), CudaError> {
        // Host-side packing: {-1→10, 0→00, 1→01}
        let packed = pack_ternary_to_u32(ternary_weights, self.n);
        self.packed_weights.copy_from_host(&packed)
    }

    /// Forward pass: y = W_ternary × x + bias
    pub fn forward(&self, activations: &[f16]) -> Result<Vec<f16>, CudaError> {
        assert_eq!(activations.len(), self.n);
        let mut d_act = GpuBuffer::alloc(self.n)?;
        d_act.copy_from_host(activations)?;

        let mut d_out = GpuBuffer::alloc(self.m)?;

        unsafe {
            crate::ffi::ternary_zero_gemv_f16(
                self.packed_weights.ptr,
                d_act.ptr as *const u16,
                d_out.ptr as *mut u16,
                self.m as i32,
                self.n as i32,
                self.stream.0,
            ).to_result()?;
            self.stream.synchronize()?;
        }

        let mut output = vec![f16::ZERO; self.m];
        d_out.copy_to_host(&mut output)?;
        Ok(output)
    }
}
```

### Step 3.3: `src/ste.rs` — Straight-Through Estimator

```rust
/// Quantize FP16 weights to ternary {-1, 0, 1} using STE-aware rounding
/// During forward: hard quantization
/// During backward: gradient passes through (STE approximation)
pub fn ternary_quantize_ste(
    weights: &[f16],        // FP16 weights [M × N]
    threshold: f32,         // Ternarization threshold (e.g., 0.5 * mean(|w|))
) -> (Vec<i8>, f32) {
    // Compute scaling factor: s = mean(|w| where |w| > threshold)
    let scale = weights.iter()
        .map(|w| w.to_f32().abs())
        .filter(|&abs_w| abs_w > threshold)
        .sum::<f32>() / weights.len() as f32;

    let ternary: Vec<i8> = weights.iter().map(|&w| {
        let wf = w.to_f32();
        if wf > threshold { 1 }
        else if wf < -threshold { -1 }
        else { 0 }
    }).collect();

    (ternary, scale)
}

/// Gradient computation with STE: dL/dw ≈ dL/dy * x (ignoring quantization)
pub fn ste_backward(
    grad_output: &[f16],    // [M]
    activations: &[f16],    // [N]
) -> Vec<f16> {
    // outer product: grad_weight[m,n] = grad_output[m] * activations[n]
    // This is the standard linear backward pass — quantization is transparent
    let m = grad_output.len();
    let n = activations.len();
    let mut grad_weights = vec![f16::ZERO; m * n];
    for mi in 0..m {
        for ni in 0..n {
            grad_weights[mi * n + ni] = grad_output[mi] * activations[ni];
        }
    }
    grad_weights
}
```

## Phase 5: Performance Analysis — Memory Wall Mitigation

### Theoretical Throughput Analysis (RTX 4060)

| Metric | FP16 GEMV | Ternary-Zero GEMV | Improvement |
|---|---|---|---|
| Bits per weight | 16 | 2 | 8× compression |
| Bytes per weight | 2 | 0.25 | 8× compression |
| Weights per 128-bit load | 8 | 64 | 8× more weights |
| Memory bandwidth utilization | 272 GB/s | 272 GB/s | Same bus |
| Effective compute bandwidth | 272 GB/s | ~2.17 TB/s (effective) | ~8× |
| Arithmetic per byte loaded | 1 FMA | ~3 conditional ops | Branchless |

### Detailed Breakdown

**1. Memory Bandwidth Dominance:**
- GEMV is memory-bound: `bytes_loaded = M × N × sizeof(weight)`
- FP16: `M × N × 2 bytes`
- Ternary-Zero: `M × N × 0.25 bytes` (2-bit packed)
- **8× reduction** in memory traffic → 8× more throughput at same bandwidth

**2. L2 Cache Effectiveness:**
- RTX 4060 has 24 MB L2 cache
- A 4096×4096 weight matrix:
  - FP16: 32 MB (doesn't fit in L2)
  - Ternary: 4 MB (fits entirely in L2!)
- **Result:** After first touch, weights served from L2 at ~500 GB/s vs 272 GB/s from DRAM

**3. Zero-Gate Skip Benefit:**
- In sparse ternary models, ~30-50% of weights are typically zero
- Zero-gate skips computation for zero weights → reduces instruction count
- Additional ~1.5-2× effective speedup beyond the memory compression

**4. Register Pressure & Occupancy:**
- 40 registers/thread target
- RTX 4060: 65536 registers/SM
- Max threads/SM at 40 regs: 65536/40 = 1638 → capped at 1536 (48 warps)
- At 256 threads/block: 6 blocks/SM → high occupancy
- **Result:** latency hiding through massive parallelism

**5. Combined Effective Throughput:**
- Theoretical DRAM bandwidth: 272 GB/s
- With 8× weight compression: effective 2.17 TB/s equivalent
- With L2 pinning (2× L2 speedup): effective ~4.3 TB/s equivalent
- With zero-skip (1.5× average): effective ~6.5 TB/s equivalent
- **For M=1, N=4096: latency ≈ (4096 × 0.25 bytes) / 272 GB/s ≈ 3.75 μs (vs 30 μs FP16)**

## Risk Mitigation

1. **PTX portability:** All PTX targets `sm_89` specifically; includes fallback C++ bitwise path via `#ifdef`
2. **FP16 precision:** Accumulation in `half2` for vectorized ops; final reduction could use `float` for accuracy
3. **Build complexity:** `build.rs` will detect CUDA_HOME / nvcc path automatically with clear error messages
4. **Memory safety:** `GpuBuffer<T>` implements `Drop` for automatic `cudaFree`; `unsafe impl Send/Sync` only for device pointers
